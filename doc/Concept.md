# Порівняльний аналіз інструментів локального Kubernetes для стартапу AsciiArtify

![Image](.data/console_demo.svg)

## Вступ

Команда AsciiArtify потребує локального Kubernetes середовища для розробки та тестування ML-продукту для перетворення зображень у ASCII-арт. У цьому документі порівнюються три популярні інструменти для локального розгортання Kubernetes: **minikube**, **kind** та **k3d**.

## 1: Основні характеристики

| Характеристика | Minikube | Kind | K3d |
| :--- | :--- | :--- | :--- |
| **Архітектура** | Віртуальна машина (або контейнери) | Контейнери Docker | Контейнери з k3s |
| **Час запуску кластера** | 2-5 хвилин | 30-60 секунд | 10-30 секунд |
| **Споживання RAM** | 2-4 GB (мінімум) | 1-2 GB | 0.5-1.5 GB |
| **Споживання CPU** | Середнє/Високе | Низьке/Середнє | Низьке |
| **Підтримка Podman** | Так (через драйвери) | Обмежена | Так (з обмеженнями) |
| **Ліцензія** | Apache 2.0 | Apache 2.0 | Apache 2.0 |

## Практичні команди порівняння

| Операція | Minikube | Kind | K3d |
| :--- | :--- | :--- | :--- |
| **Створення кластера** | `minikube start --driver=docker` | `kind create cluster --name=asciiartify` | `k3d cluster create asciiartify` |
| **Перевірка статусу** | `minikube status` | `kubectl cluster-info --context kind-asciiartify` | `k3d cluster list` |
| **Запуск Dashboard** | `minikube dashboard` | Потрібна ручна установка | Потрібна ручна установка |
| **Завантаження локального образу** | `minikube image load my-app:latest` | `kind load docker-image my-app:latest` | `k3d image import my-app:latest` |
| **Доступ до LoadBalancer** | `minikube tunnel` | Потрібен додатковий налаштування | Автоматично доступний через Traefik |
| **Видалення кластера** | `minikube delete` | `kind delete cluster --name=asciiartify` | `k3d cluster delete asciiartify` |

## Гібридна стратегія для AsciiArtify

| Етап розробки | Первинний інструмент | Додатковий інструмент | Причина |
| :--- | :--- | :--- | :--- |
| **Швидкий старт (тиждень 1)** | K3d | - | Миттєвий запуск, мінімальні налаштування |
| **Локальна розробка (тижні 2-4)** | K3d | Minikube (для GPU тестів) | Швидкі ітерації + тестування з GPU |
| **CI/CD налаштування (місяць 2)** | Kind | - | Стабільність в CI, декларативна конфігурація |
| **Продакшн-подібне тестування (місяць 3)** | Minikube | Kind | Повна сумісність з production |
| **Фінальна валідація (перед релізом)** | Minikube | Kind + K3d | Комплексне тестування на всіх платформах |

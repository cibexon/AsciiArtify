# Порівняльний аналіз інструментів локального Kubernetes для стартапу AsciiArtify

## Вступ
Команда AsciiArtify потребує локального Kubernetes середовища для розробки та тестування ML-продукту для перетворення зображень у ASCII-арт. У цьому документі порівнюються три популярні інструменти для локального розгортання Kubernetes: **minikube**, **kind** та **k3d**.

## Таблиця 1: Основні характеристики

| Характеристика | Minikube | Kind | K3d |
| :--- | :--- | :--- | :--- |
| **Архітектура** | Віртуальна машина (або контейнери) | Контейнери Docker | Контейнери з k3s |
| **Час запуску кластера** | 2-5 хвилин | 30-60 секунд | 10-30 секунд |
| **Споживання RAM** | 2-4 GB (мінімум) | 1-2 GB | 0.5-1.5 GB |
| **Споживання CPU** | Середнє/Високе | Низьке/Середнє | Низьке |
| **Підтримка Podman** | Так (через драйвери) | Обмежена | Так (з обмеженнями) |
| **Ліцензія** | Apache 2.0 | Apache 2.0 | Apache 2.0 |
